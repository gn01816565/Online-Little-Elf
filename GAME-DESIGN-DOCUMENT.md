# 🎮 線上多人小精靈遊戲 - 完整設計規劃文件

## 📋 文件資訊
- **專案名稱**: Online Little Elf / 線上多人小精靈
- **目標平台**: Web (多裝置支援)
- **遊戲類型**: 多人線上競技 / 休閒社交
- **目標玩家**: 10-50歲，適合派對、攤位、線上聚會
- **最大玩家數**: 10人同時在線
- **版本**: v2.0 規劃

---

## 🎯 核心設計理念

### 願景
打造一款**易上手、高互動、可重玩性強**的多人小精靈遊戲，適合朋友聚會、線上社交、電競賽事等多種場景。

### 設計三大支柱
1. **即時競技** - 快節奏的多人對抗體驗
2. **社交互動** - 強化玩家間的互動與連結
3. **持續成長** - 提供長期遊玩動力

---

## 📊 當前狀態分析

### ✅ 現有優勢
- 支援10人同時在線的技術架構
- Firebase 即時同步機制穩定
- 基礎房間系統運作良好
- 簡潔清晰的 UI/UX
- 支援繁體中文/英文雙語

### ❌ 需要改進的問題
- **缺乏明確的遊戲目標**：沒有勝負判定機制
- **玩家互動不足**：只是各自收集豆子，缺乏競爭
- **缺乏遊戲深度**：玩法單一，可重玩性低
- **無社交粘性**：沒有好友、排行榜等社交功能
- **缺少成長系統**：無等級、解鎖、收集要素
- **視聽反饋不足**：缺少音效、粒子特效等

## 🎨 第一層：核心遊戲循環強化

### 1. 遊戲模式系統

#### 1.1 經典模式
```
📌 計時競速賽（Time Attack）
- 遊戲時長: 3分鐘
- 勝利條件: 時間結束時分數最高
- 得分機制:
  * 小豆子: 10分
  * 能量豆: 50分
  * 連擊加成: 連續吃豆不間斷可獲得 x1.5 / x2.0 倍率

📌 清空挑戰（Clear Challenge）
- 勝利條件: 最快吃完所有豆子
- 計時機制: 顯示個人用時
- 排名: 依完成時間排序

📌 生存模式（Survival Mode）
- AI 幽靈追殺: 3-5個AI幽靈在地圖遊蕩
- 生命值: 每位玩家3條命
- 勝利條件: 最後存活者獲勝
- 能量豆效果: 吃到後可反殺幽靈（30秒）
```

#### 1.2 競技模式
```
📌 個人競賽（Free For All）
- 2-10人混戰
- Elo 排名系統
- 賽季積分累積
- 段位晉升機制

📌 團隊對抗（Team Battle）
- 紅隊 vs 藍隊（最多5v5）
- 團隊總分競賽
- 團隊技能協同（如：隊友可互相傳送）
- 語音頻道分組

📌 獵人模式（Hunter Mode）
- 1個獵人 vs 9個逃亡者
- 獵人擁有特殊能力（加速、追蹤）
- 逃亡者需生存3分鐘
- 獵人每抓到1人獲得強化
```

#### 1.3 合作模式
```
📌 PvE 模式（Co-op vs AI）
- 所有玩家 vs AI 幽靈大軍
- 波次系統: 15波逐漸增強的敵人
- 共享生命池
- 合作得分加成

📌 Boss 戰（Boss Battle）
- 團隊合作擊敗巨型 Boss
- Boss 技能:
  * 召喚小兵
  * 範圍攻擊
  * 地圖改變
- 擊敗獎勵稀有皮膚/道具
```
