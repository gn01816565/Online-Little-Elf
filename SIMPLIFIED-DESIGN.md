# 🎮 線上多人小精靈 - 極簡版設計（推薦方案）

## 💡 設計哲學

**一句話總結**：10個人在同一個迷宮裡吃豆子，搶能量豆，互相追逐，3分鐘後分數最高者獲勝。

### 設計原則
```
✅ 5 秒看懂規則
✅ 0 秒教學（直接開始玩）
✅ 3 分鐘一局（快節奏）
✅ 10 人歡樂（適合派對）
✅ 無需成長系統（進來就玩）
```

---

## 🎮 遊戲規則（極簡版）

### 唯一目標
**3 分鐘內收集最多分數**

### 得分方式
```
⚪ 小豆子：10 分
💎 能量豆：50 分（全地圖固定 4 個）
👊 碰撞其他玩家：搶 20% 分數
🌟 吃到能量豆後追到玩家：搶 50% 分數
```

### 就這樣！沒了！

## 💻 技術實作（極簡版）

### 需要開發的功能

#### ✅ 必做（核心玩法）
```
1. 基礎迷宮地圖 (2-3小時)
   - 用現有的地圖生成即可
   - 不需要很大

2. 豆子收集 (1小時)
   - 已經有了，復用即可

3. 能量豆機制 (3-4小時)
   - 吃到後變身 15 秒
   - 視覺：閃爍 + 變大
   - 其他玩家看到會逃跑

4. 碰撞搶分 (2-3小時)
   - 普通碰撞：搶 20%
   - 能量豆碰撞：搶 50%
   - 分數飄字動畫

5. 倒數計時 (1小時)
   - 3分鐘倒數
   - 最後30秒警告

6. 結算畫面 (2小時)
   - 顯示前3名
   - 再玩一局

總計：11-14 小時 ≈ 2天完成！
```

#### ❌ 不做（過度設計）
```
❌ AI 幽靈（太複雜）
❌ 攻擊/防禦按鈕（不需要）
❌ 裝備系統（不需要）
❌ 等級系統（不需要）
❌ 成就系統（不需要）
❌ 複雜 UI（不需要）
❌ 技能樹（不需要）
❌ 大逃殺（太複雜）
```

如果達成 → 成功！
如果沒達成 → 調整後再測試

## 🎮 為什麼這個設計比複雜版本好？

### 1. **快速驗證**
```
複雜版本：5-7 天才知道好不好玩
極簡版本：3 天就知道

→ 節省時間，快速迭代
```

### 2. **容易調整**
```
複雜版本：改一個數值影響一堆系統
極簡版本：只有幾個參數要調

→ 平衡性容易掌握
```

### 3. **玩家友善**
```
複雜版本：新手要學很多
極簡版本：看一眼就會玩

→ 玩家容易上手
```

### 4. **技術穩定**
```
複雜版本：50人、裝備、AI... 容易出 Bug
極簡版本：只有基礎功能，穩定

→ 遊戲體驗好
```

### 5. **社交屬性強**
```
複雜版本：需要練習、學習策略
極簡版本：朋友聚會立刻開玩

→ 適合派對、攤位
```
